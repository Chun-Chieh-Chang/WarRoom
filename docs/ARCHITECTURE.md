# 系統架構與設計

## 1. 架構模式：飛輪 (The Flywheel)

本系統避免使用混亂的單體代碼，轉而採用由獨立原子單元組成的架構，這些單元彼此相互強化。

### 分層結構

1.  **核心領域邏輯 (核心 Hub)**
    - 純函數、業務規則、不變量。
    - _依賴_：無。
2.  **接口適配器 (輻條 Spokes)**
    - API 用戶端、UI 組件、CLI 處理程序。
    - _依賴_：依賴於核心領域。

3.  **基礎設施與工具 (輪緣 Rim)**
    - 資料庫、外部 API、框架。
    - _依賴_：作為插件插入系統。

## 2. 目錄結構 (MECE 原則)

專案結構遵循「相互獨立、完全窮盡」(MECE) 原則，以在專案增長時保持秩序。

```
CompoundEffect/
├── docs/               # 大腦：架構、SOP、日誌
│   ├── DEV_LOG.md      # 開發線性歷史
│   ├── ARCHITECTURE.md # 設計演進
│   ├── PROCESS.md      # 開發流程閉環
│   └── REAL_WORLD_EXAMPLES.md # 生活實例對應
├── src/                # 肌肉：原始碼
│   ├── core/           # 業務邏輯 (純引擎)
│   ├── lib/            # 共享工具 (複合資產)
│   └── components/     # 可複用的 UI/邏輯塊
├── tests/              # 安全網：自動化驗證
└── .github/            # 自動化：CI/CD 工作流
```

## 3. 擴展計劃

- **第一階段**：手動循環。建立核心工具與手動測試。
- **第二階段**：自動化循環。引入 CI/CD 與單元測試。
- **第三階段**：數據循環。引入指標與分析來驅動決策。
