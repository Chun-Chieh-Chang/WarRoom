# 複利效應系統 (Compound Effect System) 使用指南

歡迎使用 **Flywheel V6 (Constitutional Edition)**。本系統是一個基於「複利效應」哲學設計的自主增長引擎，結合了 AI 決策內閣、多資產協同以及抗脆弱機制。

---

## 核心概念 (Core Concepts)

### 1. 蓄能池 (Assets)

系統目前支援兩類核心資產：

- **增長資產 (Growth Asset)**：代表數值化的價值（如財富、體能、用戶數）。
- **知識資產 (Knowledge Asset)**：代表智慧的累積。知識資產的版本與深度會直接影響增長資產的效率。

### 2. 五步飛輪 (The Flywheel)

系統每一秒（或每一週期）都會執行一個完整的閉環：

1. **感知 (Sense)**：檢查當前環境（是否存在黑天鵝）與資產協同乘數。
2. **決策 (Think)**：啟動 AI 內閣辯論，參考「系統憲法」做出最佳判斷。
3. **執行 (Act)**：應用 INVEST（投入）或 COMPOUND（複利）動作。
4. **反省 (Reflect)**：分析執行結果並轉化為經驗。
5. **儲存 (Remember)**：將所有紀錄寫入 SQLite 資料庫。

### 3. 智慧內閣 (Cabinet Consensus)

為了確保決策的高品質，系統引入了三個 AI 角色：

- **提案者 (Proposer)**：專注於激進增長。
- **審查者 (Auditor)**：專注於風險控管與歷史教訓。
- **主席 (Chair)**：權衡兩方意見，做出最終執行決定。

### 4. 系統憲法 (Constitution)

每運作 10 個週期，系統會從過去的原始經驗中「提煉」出 3 條高階導向原則。這些原則會存入憲法，作為未來 AI 決策的最高指導準則。

---

### 1. 啟動互動儀表板

啟動網頁伺服器：

```bash
npm run dashboard
```

開啟瀏覽器訪問：`http://localhost:3000`

### 2. 觸發系統自轉 (Manual Flywheel Trigger)

在儀表板右上角點擊 **「啟動下一週期 [FLY]」** 按鈕。

- 點擊後，AI 會立即開始「感知、決策、執行、反省」的飛輪運轉。
- 網頁圖表與日誌會在運轉完成後自動刷新。

### 3. CLI 手動執行 (進階)

如果您想繞過 Web 界面或進行批次執行，可以在終端機執行：

```bash
npx ts-node src/main.ts
```

## 界面說明 (Dashboard Overview)

- **資產增長曲線**：顯示增長資產隨時間的變化。可以看到複利帶來的指數增長。
- **決策分佈圖**：統計 AI 選擇「投入」與「複利」的比例。
- **智慧累積列表**：顯示最近的 AI 自我反省與學習到的教訓。
- **審計日誌 (Action Log)**：詳細記錄每一步數值變動的前後值、變動幅度，以及環境衝擊（黑天鵝）。
- **系統憲法 (System Constitution)**：顯示目前系統最高版本的引導原則。

---

## 進階機制

- **黑天鵝 (Black Swan)**：系統有 10% 的機率遭遇外部變動。危機時會導致資產縮水，橫財時會大幅增加資產。系統會從中學習「抗脆弱性」。
- **協同效應 (Synergy)**：當您的知識越豐富，資本增長的乘數就越高（最高可達 2.5x）。
