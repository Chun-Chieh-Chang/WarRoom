# 複利開發流程

本文件概述了構建「複利效應系統」的操作閉環。每一次的變動都必須遵循此循環，以確保價值的積累。

## 五步閉環流程

1.  **定義資產 (蓄能池 Accumulator)**
    - _關鍵問題_：我現在構建的什麼資產在未來可以被再次使用？
    - _產出_：在 `src/core` 中定義清晰的接口或數據結構。

2.  **編寫測試 (連接器 Connector)**
    - _關鍵問題_：如果不進行手動檢查，我該如何確保這個資產持續運作？
    - _產出_：在 `tests/` 中編寫一個失敗的測試案例。

3.  **實作功能 (核心引擎 Engine)**
    - _關鍵問題_：讓測試通過的最簡單路徑是什麼？
    - _產出_：在 `src/` 中編寫功能代碼。

4.  **重構優化 (減阻設計 Friction Reducer)**
    - _關鍵問題_：這段代碼是否易於閱讀且方便下一次循環修改？
    - _產出_：清理、格式化並加入註釋的代碼。

5.  **紀錄日誌 (知識庫 Knowledge Base)**
    - _關鍵問題_：我學到了什麼是其他人（或未來的我）需要知道的？
    - _產出_：更新 `docs/DEV_LOG.md`。

## 自動化規則

- **提交前檢查**：`npm test` 與 `npm run lint` 必須通過。
- **文檔維護**：`README.md` 與 `ARCHITECTURE.md` 是活文件，必須隨系統演進更新。
