# CompoundEffect 專案憲章

## 1. 願景

建立一個能夠展現 **「複利效應」（正向強化循環）** 的軟體系統，讓每一次的迭代、使用者互動與每一行代碼，都能隨時間推移，為系統價值與效用帶來指數級的增長。

## 2. 核心哲學

專案架構嚴格遵循 **「飛輪系統」（Flywheel System）** 原則：

1.  **積累 (Accumulation)**：每一次的輸出都必須轉化為下一輪循環的「資產」（數據、組件或知識）。
2.  **模組化 (Modularity)**：建立獨立且可複用的組件，以降低未來功能開發的邊際成本。
3.  **自動化 (Automation)**：重度投入 CI/CD 與測試，防止功能倒退（摩擦力），確保動能始終向前。

## 3. 系統架構維度

### A. 核心引擎 (價值)

- **目標**：以最小的摩擦力解決使用者的即時問題。
- **指標**：價值實現時間 (Time-to-Value, TTV)。

### B. 蓄能池 (資產)

- **數據**：使用者行為與輸入不斷優化模型與邏輯。
- **代碼庫**：一個高內聚、低耦合的模組庫。
- **知識**：`DEV_LOG.md` 紀錄所有上下文，防止「知識債」。

### C. 反饋閉環 (優化)

- 自動化測試與錯誤報告 -> 即時提升代碼品質。
- 使用者數據指標 -> 優化功能優先順序。

## 4. 開發標準 (SOP)

- **精準修改**：僅針對必要部分進行修訂，在修復前必須先診斷根本原因。
- **測試驅動**：無測試驗證（Console 無錯誤）則功能不算完成。
- **文檔同步**：目錄結構與日誌必須隨代碼變動同步更新。
